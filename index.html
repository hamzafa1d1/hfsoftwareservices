02 / WORK

Selected work

Rebuilding a compensation platform with AI agents & spec-driven development

Context: legacy compensation management system requiring a full rewrite, delivered with a fully AI-integrated engineering workflow.

Problem
The existing system was too brittle to extend and too risky to refactor incrementally — it needed to be replaced without losing years of embedded business logic, and delivered fast.
What I did
Rebuilt the platform from scratch using spec-driven development — writing detailed functional and technical specs first, then using AI coding agents (GitHub Copilot) to accelerate implementation against those specs. Wired GitHub CI/CD so every pull request gets automatic AI code review and can auto-merge when checks pass. Connected Jira and Confluence via MCP so tickets were created, updated, and documented automatically from the development workflow — keeping the board and wiki in sync without manual overhead. The result was a clean .NET backend with well-defined domain boundaries and a fully traceable, documented codebase.
Result
Full platform delivered significantly faster than a conventional rewrite, with automated test coverage, living documentation in Confluence, and a Jira board that stayed current throughout — without any manual ticket management.

Catching fraud in near-real-time

Context: Mastercard transaction stream at a corporate-spend platform.

Problem
Suspicious transactions were caught too slowly, and manual review didn't scale.
What I did
Built a multi-layer, agentic fraud-detection system combining layered LLM analysis to flag activity in near-real-time — .NET backend with an Angular review UI.
Result

~79% detection accuracy in production, manual reviews halved, €3,000+ in fraudulent charges recovered — all on a .NET backend with no dedicated ML team.

Migrating a live database with zero downtime

Context: legacy codebase being moved to a Domain-Driven Design architecture.

Problem
A complex schema migration had to happen on a production system that couldn't go down.
What I did
Authored cautious SQL scripts to avoid resource locking, prepared and monitored migration jobs with detailed logging to catch failures early.
Result
Completed with zero downtime and <2% CPU overhead during peak load.

Making payments provider-agnostic

Context: a payments layer tightly coupled to one processor.

Problem
Adding or swapping payment providers was slow and risky.
What I did
Refactored the payments layer to be provider-agnostic and integrated Adyen cleanly, applying Observer / Factory / Decorator patterns for maintainability.
Result
~30% faster feature delivery and ~20% faster onboarding for new work on the system.

Also: Mastercard virtual-card renewal automation (100+ cards renewed before expiry, protecting interchange revenue) and an automated credit-card suspension feature serving 1,000+ corporate users.
